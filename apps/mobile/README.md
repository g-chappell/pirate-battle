# @pirate-battle/mobile

Capacitor wrap of `apps/web` for native Android delivery.

**Status:** Android scaffold present (TASK-042). Wallet UX (CIP-45 /
WalletConnect bridge) lands separately under TASK-043.

## Layout

- `capacitor.config.ts` — `webDir = "../web/dist"`, `appId =
  "dev.blacksail.piratebattle"`, `appName = "Pirate-Battle"`.
- `android/` — generated by `npx cap add android`. Committed. Gradle
  build outputs (`build/`, `.gradle/`, `app/build/`, `local.properties`)
  are gitignored at the repo root; Capacitor's own `android/.gitignore`
  also covers `cap sync` outputs (`app/src/main/assets/public/`, the two
  generated `capacitor.*.json` files).

iOS is intentionally out of scope: the autonomous pipeline runs on a
Linux VPS with no macOS access, and `npx cap add ios` requires
macOS + Xcode + CocoaPods. Revisit only if/when a macOS runner becomes
part of the pipeline.

## Commands

```bash
# from repo root
npm run build  -w @pirate-battle/mobile   # builds web + cap sync android
npm run sync   -w @pirate-battle/mobile   # cap sync android only
npm run open:android -w @pirate-battle/mobile   # opens in Android Studio (local)
```

`build` chains `npm run build -w @pirate-battle/web` then `cap sync
android` — the web `dist/` is the source the native shell loads.

## CI

CI does not yet run a Gradle build of the Android project; the scaffold
is verified by `npx cap sync android` exiting 0 (which Capacitor does
without invoking Gradle). A device/emulator build can be added later
once an Android SDK is provisioned on the runner.
